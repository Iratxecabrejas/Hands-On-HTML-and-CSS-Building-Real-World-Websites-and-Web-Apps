Variable Declarations & Accidental Globals

1. Assigning without Declaration (Non-strict Mode)


// This creates a global variable unintentionally!
function makeGlobal() {
  accidentalGlobal = 123; // no var/let/const!
  console.log("accidentalGlobal inside function:", accidentalGlobal);
}
makeGlobal();
console.log("accidentalGlobal outside function:", accidentalGlobal);

2. Function Parameter is Not Global


// Function arguments never create global variables
function paramDemo(param) {
  param = 55;
  console.log("param inside function:", param);
}
paramDemo(44);
try {
  console.log(param); // ReferenceError!
} catch(e) {
  console.log("Error:", e);
}

3. Strict Mode: No Accidental Globals


"use strict";
function strictGlobalDemo() {
  try {
    noVarHere = 'oops'; // Throws ReferenceError!
  } catch(e) {
    console.log("Error:", e.message);
    alert("Error: " + e.message);
  }
}
strictGlobalDemo();
Open your browser console to see console.log output. All alerts summarize the key results.
Function parameters never become global; accidental globals only happen through assignments without declaration outside strict mode.
