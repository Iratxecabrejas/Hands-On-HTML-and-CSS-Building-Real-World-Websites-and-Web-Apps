3. OR Default (||)

Syntax: value || default
Returns the left value, unless it's falsy (undefined, null, 0, "", false, NaN), then it returns the default instead. 

let username;
let displayName = username || "Stranger";
console.log(displayName); // Outputs: "Stranger"
Outputs: Stranger

4. Nullish Coalescing (??)

Syntax: value ?? default
Returns the left value unless it is null or undefined (only!), otherwise returns the default.
Great for handling *real* "missing" data while allowing 0, "", or false as valid. 

let highScore = 0;
let displayScore = highScore ?? "No Score Yet";
console.log(displayScore); // Outputs: 0 (not defaulted!)
let unknown;
let safe = unknown ?? "Safe fallback";
console.log(safe); // Outputs: "Safe fallback"
Outputs: displayScore: 0, safe: Safe fallback

Compare: || vs ??

|| treats all falsy values as "default".
?? only treats null and undefined as "default". 

let inputCatName = "";
let catNameOr = inputCatName || "Mr. Fluffy";    // "Mr. Fluffy" (because "" is falsy)
let catNameNull = inputCatName ?? "Mr. Fluffy";  // "" (empty string is valid)

console.log("catNameOr:", catNameOr);
console.log("catNameNull:", catNameNull);
